# SecuRisk — Система управления ИБ (GRC) 🔒

**SecuRisk** — это современная платформа для автоматизации процессов информационной безопасности, разработанная в соответствии со стандартом **ISO 27001**. Система позволяет вести реестры активов, рисков и инцидентов, обеспечивая полный контроль над состоянием ИБ в организации.

## 🌟 Ключевые возможности

* **Dashboard**: Мониторинг ключевых метрик ИБ (MTTA, MTTR, уровни рисков) в реальном времени.
* **Реестр рисков**: Идентификация, оценка и мониторинг обработки рисков.
* **Управление инцидентами**: Регистрация и расследование инцидентов ИБ с автоматическим расчетом метрик эффективности.
* **Реестр активов**: Классификация и учет информационных активов организации.
* **Wiki & Регламенты**: Внутренняя база знаний для хранения политик и процедур.
* **Управление пользователями**: Гибкая ролевая модель доступа (RBAC).

## 🛠 Технологический стек

* **Frontend**: React 19, Tailwind CSS, Shadcn/UI, Craco.
* **Backend**: FastAPI (Python 3.12), Motor, Pydantic.
* **Database**: MongoDB 7.0.
* **Infrastructure**: Nginx (Reverse Proxy), Supervisor (Process Manager).

## 🚀 Развертывание (Ubuntu 24.04+)

Система устанавливается полностью автоматически. Скрипт настроит базу данных, создаст администратора, соберет фронтенд и настроит веб-сервер.

```bash
# 1. Клонируйте репозиторий
git clone <url_репозитория>
cd securisk

# 2. Установите систему
sudo chmod +x *.sh
sudo ./install.sh

```

> **Учетные данные после установки:**
> * **Логин**: `admin`
> * **Пароль**: (задается вами в процессе установки).

## 🔄 Эксплуатация и разработка

### Применение изменений

Если вы внесли изменения в код фронтенда или бэкенда, запустите скрипт обновления. Он пересоберет статику и перезапустит сервисы:

```bash
sudo ./update.sh

```

### Управление процессами

Система использует **Supervisor** для обеспечения бесперебойной работы.

```bash
sudo supervisorctl status             # Статус сервисов
sudo supervisorctl restart securisk-backend  # Рестарт API
sudo supervisorctl tail -f securisk-backend  # Логи API

```

### Логирование

* **API ошибки**: `/var/log/supervisor/securisk-backend.err.log`
* **Nginx доступ**: `/var/log/nginx/access.log`

## 🛡 Безопасность

1. **JWT Auth**: Вся работа с API защищена токенами.
2. **Bcrypt**: Пароли хранятся в виде надежных хешей (bcrypt 3.2.2).
3. **MongoDB Auth**: Доступ к базе данных защищен паролем на уровне системного пользователя.
4. **RBAC**: Доступ к модулям ограничен ролями (Администратор, Инженер, Специалист).
